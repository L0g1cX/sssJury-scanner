# sssJury

`sssJury.py` 是一个用于资产整理、存活探测、指纹识别、优先级评分与报告导出的命令行工具。它面向安全测试、资产梳理和渗透前期目标筛选场景，可以从多个输入文件中提取目标 URL，调用 `httpx` 进行批量探测，再根据页面标题、技术栈、状态码、URL 特征等信息自动打分排序，最终导出 CSV 和 HTML 报告。

---

## 一、工具简介

`sssJury.py` 的主要用途是：

- 从 `txt` / `csv` 文件中提取资产目标
- 自动识别并去重 URL
- 对目标执行 `httpx` 批量存活探测
- 收集标题、技术栈、状态码等指纹信息
- 根据内置规则对资产进行风险/价值评分
- 按分数降序排序，输出优先级目标清单
- 导出 `Prioritized_Targets_Report.csv`
- 导出 `Prioritized_Targets_Report.html`

该工具适合在资产测绘、攻击面梳理、渗透测试前置筛选、漏洞验证目标优先级排序等场景中使用。

---

## 二、核心功能

### 1. 输入文件读取与目标提取

工具支持两种输入方式：

- **手动输入文件路径**
  - 支持多个文件，使用英文逗号分隔
- **自动匹配当前目录文件**
  - 自动查找当前目录中的：
    - `R1.txt`
    - `R1.csv`
    - `R2.txt`
    - `R2.csv`
    - 依次向后递增

工具会从这些文件中提取目标地址，并尽量识别以下格式：

- `http://example.com`
- `https://example.com`
- `example.com`
- `1.2.3.4`
- `1.2.3.4:8080`
- `example.com/path`

同时会过滤掉明显不是 URL 的普通文本内容。

### 2. URL 去重

工具会对提取到的 URL 进行集合去重，避免重复扫描同一目标，提高效率。

### 3. 状态码预判跳过

在输入记录中，如果某条目标旁边显式包含以下状态码：

- `404`
- `502`
- `503`

则该目标会被认为是低优先级或不可用目标，不会送入 `httpx` 探测，而是直接放到最终结果末尾，作为跳过项展示。

### 4. 调用 httpx 批量探测

工具会调用外部程序 `httpx` 对目标进行批量探测，输出包括：

- 状态码
- 页面标题
- 技术栈识别
- 服务信息
- JSON 格式结果

支持分批、并发执行，以提高大规模资产探测效率。

### 5. 优先级评分

工具内置了一套评分规则，会根据以下维度给目标加分：

- URL 中是否包含敏感关键字
- 页面标题是否含有高价值入口词
- 技术栈是否命中高危框架
- 是否存在默认页面、中间件特征
- 是否存在敏感路径
- 是否使用非标准端口
- 是否返回 `200` / `401` / `403` 等状态码

评分规则可以帮助你快速从海量资产中筛选出更值得优先关注的目标。

### 6. 结果排序

所有探测结果会按照分数从高到低排序，方便直接查看最值得优先处理的资产。

### 7. CSV 导出

工具会生成：

- `Prioritized_Targets_Report.csv`

CSV 中包含以下字段：

- `Score`
- `URL`
- `Title`
- `Status`
- `Technologies`
- `Reasons`

### 8. HTML 报告导出

工具还会生成一个同名 HTML 报告：

- `Prioritized_Targets_Report.html`

HTML 报告具有以下特点：

- 表格化展示结果
- 支持列宽拖拽
- 带有基础样式美化
- 适合浏览器中直接查看
- 便于快速筛选与人工复核

## 三、输出文件说明

### 1. `Prioritized_Targets_Report.csv`

这是最终的结构化结果文件，适合：

- 二次处理
- 导入表格软件
- 批量分析
- 做后续数据归档

### 2. `Prioritized_Targets_Report.html`

这是可视化报告文件，适合：

- 浏览器查看
- 展示给团队成员
- 快速筛选高价值资产

### 3. `httpx_output.json`

这是 `httpx` 的中间结果文件，用于解析和评分。

### 4. `httpx_temp/`

这是运行时的临时目录，存放分批目标文件和分批扫描结果，任务完成后通常会自动清理。

## 四、评分规则说明

当前脚本的评分逻辑主要由以下规则构成：

### 1. URL 敏感关键字

如果 URL 中包含如下关键字，会加分：

- `test`
- `dev`
- `uat`
- `backup`
- `bak`
- `oa`
- `git`
- `svn`
- `jenkins`
- `api`
- `admin`
- `swagger`
- `actuator`

### 2. 高危框架特征

如果页面标题或技术栈中包含如下框架，会加分：

- `struts`
- `thinkphp`
- `shiro`
- `fastjson`
- `weblogic`
- `jboss`
- `spring`
- `ruoyi`
- `laravel`
- `drupal`

### 3. 高价值标题特征

如果标题中出现如下词汇，会加分：

- `登录`
- `管理`
- `后台`
- `系统`
- `login`
- `admin`
- `dashboard`
- `platform`
- `中心`
- `入口`

### 4. 弱配置/默认页特征

如果标题或技术栈中出现如下特征，会加分：

- `welcome to nginx`
- `apache tomcat`
- `iis windows`
- `directory listing`
- `index of`

### 5. 敏感路径

如果 URL 中包含如下路径，会加分：

- `/api`
- `/admin`
- `/actuator`
- `/swagger`
- `/manage`
- `/console`
- `/backend`
- `/system`

### 6. 非标准端口

如果目标使用的是非 `80` / `443` 端口，会额外加分。

### 7. 状态码因素

- `200`：额外加分
- `401` / `403` 且命中敏感路径：额外加分
- `401` / `403` 但未命中敏感路径：仅记录原因

## 五、运行流程

完整执行流程如下：

1. 读取输入文件
2. 提取 URL 并去重
3. 识别带 `404` / `502` / `503` 的跳过项
4. 分批调用 `httpx`
5. 收集返回结果
6. 根据规则计算分数
7. 排序
8. 导出 CSV
9. 导出 HTML
